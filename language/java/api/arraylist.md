### ArrayList没有自动缩容
`ArrayList`底层是有一个`Object`类型的数组,默认初始化大小为10(第一次添加元素),后续扩容为原来数组的1.5倍,删除中间的元素,会将后续的元素拷贝到前面的位置,再将最后的位置置为`null`
因为没有自动缩容,所以当底层数组很大的时候,在删除前面元素之后,这个拷贝会有一定的影响(是通过`System.arraycopy`进行拷贝的,底层是`c\c++`实现的),如果想主动缩容,可以调用`trimToSize`方法,
至于原因,在于业界对于空间和性能的考量,对于扩容的需求,很明显缩容的场景不是很必要,并且频繁的变动底层数组在性能上确实会收到影响
